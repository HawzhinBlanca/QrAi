# Web Accessibility Baseline Plan

1. Add keyboard skip navigation from the app shell to the main workspace.
2. Add stable accessible names to login/register controls without changing the visual form layout.
3. Expose login/register failures as alert content.
4. Expose sidebar active state through ARIA.
5. Add CSS fallbacks for focus visibility, touch targets, and reduced-motion preferences.
6. Verify with focused web tests/build, then the canonical repository gate.
